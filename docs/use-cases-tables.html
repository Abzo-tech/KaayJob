Tableau 6 : Cas d’utilisation - S’inscrire sur la plateforme

Élément Description
Nom du cas d’utilisation S’inscrire sur la plateforme
Acteurs Client, Prestataire
Déclencheur L’acteur sélectionne l’option d’inscription depuis l’interface d’authentification.
Préconditions L’acteur n’est pas encore authentifié et accède à la page d’accueil ou à la page de connexion.
Description / Flux principal 1. L’acteur ouvre le formulaire d’inscription.
2. Le système affiche les champs requis.
3. L’acteur renseigne ses informations personnelles : prénom, nom, email, téléphone et mot de passe.
4. L’acteur sélectionne son rôle : client ou prestataire.
5. L’acteur valide le formulaire.
6. Le système contrôle la validité des données saisies.
7. Le système enregistre le nouvel utilisateur dans la base de données.
8. Si l’acteur est un prestataire, le système prépare également son espace professionnel.
9. Le système authentifie l’utilisateur puis le redirige vers l’interface correspondant à son rôle.
Scénarios alternatifs / Exceptions Email déjà utilisé : le système rejette l’inscription et affiche un message d’erreur.
Champs invalides ou incomplets : le système signale les erreurs de validation.
Rôle incorrect : l’inscription est refusée.
Post-conditions L’utilisateur dispose d’un compte valide et peut accéder à la plateforme selon son rôle.

Tableau 7 : Cas d’utilisation - Rechercher un prestataire

Élément Description
Nom du cas d’utilisation Rechercher un prestataire par catégorie ou localisation
Acteurs Visiteur, Client
Déclencheur L’acteur consulte les catégories de services ou accède à la liste des prestataires.
Préconditions L’acteur se trouve sur la page d’accueil, la page des catégories ou la page de recherche des prestataires.
Description / Flux principal 1. L’acteur parcourt les catégories de services disponibles.
2. Il sélectionne une catégorie ou un type de prestation.
3. Le système affiche les prestataires correspondants sous forme de liste.
4. L’acteur peut également consulter une représentation cartographique des prestataires.
5. Il affine éventuellement sa recherche selon les options disponibles dans l’interface.
6. Il sélectionne un prestataire ou un service.
7. Le système affiche la fiche détaillée du service ou du prestataire choisi.
Scénarios alternatifs / Exceptions Aucun résultat trouvé : le système invite l’acteur à modifier ses critères de recherche.
Localisation indisponible : la consultation reste possible via la liste.
Erreur de chargement : un message d’information est affiché.
Post-conditions L’acteur identifie un ou plusieurs prestataires et peut consulter leurs informations détaillées.

Tableau 8 : Cas d’utilisation - Créer un profil prestataire

Élément Description
Nom du cas d’utilisation Créer et compléter un profil professionnel
Acteurs Prestataire
Déclencheur Le prestataire accède à la rubrique « Mon profil » depuis son espace personnel.
Préconditions L’utilisateur est authentifié avec le rôle de prestataire.
Description / Flux principal 1. Le prestataire ouvre son espace de profil.
2. Le système affiche le formulaire de mise à jour des informations professionnelles.
3. Le prestataire renseigne les données demandées : spécialité, description, zone d’intervention, localisation, tarifs et autres informations utiles.
4. Il complète, si nécessaire, ses disponibilités.
5. Il valide les modifications.
6. Le système enregistre les informations mises à jour dans la base de données.
7. Le prestataire peut ensuite soumettre une demande de vérification.
8. L’administrateur examine la demande.
9. Si la demande est approuvée, le profil est identifié comme vérifié.
Scénarios alternatifs / Exceptions Informations incomplètes : le système demande les corrections nécessaires.
Données invalides : l’enregistrement est refusé.
Demande de vérification rejetée : le profil reste actif mais non vérifié.
Post-conditions Le profil professionnel du prestataire est enregistré et exploitable par la plateforme.

Tableau 9 : Cas d’utilisation - Réserver un service

Élément Description
Nom du cas d’utilisation Effectuer une réservation de service
Acteurs Client
Déclencheur Le client sélectionne l’option de réservation depuis la fiche d’un service.
Préconditions Le client est authentifié et consulte le détail d’un service.
Description / Flux principal 1. Le client accède à la fiche détaillée d’un service.
2. Il clique sur l’option de réservation.
3. Le système affiche le formulaire correspondant.
4. Le client saisit les informations nécessaires : date, heure, adresse, ville, téléphone et notes éventuelles.
5. Il soumet sa demande.
6. Le système enregistre la réservation avec un statut initial en attente.
7. Le système transmet l’information au prestataire concerné.
8. Le prestataire consulte la demande depuis son espace.
9. Il peut confirmer, refuser, démarrer ou clôturer la prestation.
10. Le client suit l’évolution de sa réservation depuis son tableau de bord.
Scénarios alternatifs / Exceptions Données invalides : le système affiche les erreurs de saisie.
Service indisponible : la réservation n’est pas créée.
Réservation rejetée : le client est informé du refus.
Réservation annulée : le statut est mis à jour dans le système.
Post-conditions Une réservation est créée, enregistrée et suivie dans la plateforme.

Tableau 10 : Cas d’utilisation - Noter un prestataire

Élément Description
Nom du cas d’utilisation Évaluer une prestation et publier un avis
Acteurs Client
Déclencheur Le client sélectionne l’option « Laisser un avis » sur une prestation terminée.
Préconditions Le client dispose d’une réservation terminée qui n’a pas encore été évaluée.
Description / Flux principal 1. Le client accède à son espace personnel.
2. Il consulte l’historique de ses réservations terminées.
3. Il choisit la prestation à évaluer.
4. Il attribue une note.
5. Il saisit un commentaire décrivant son appréciation.
6. Il valide son évaluation.
7. Le système enregistre l’avis dans la base de données.
8. Le système rattache cet avis à la réservation, au service et au prestataire concernés.
9. L’évaluation devient visible dans les espaces de consultation associés.
Scénarios alternatifs / Exceptions Tentative de double évaluation : le système refuse l’opération.
Réservation non terminée : l’évaluation n’est pas autorisée.
Données invalides : un message d’erreur est affiché.
Post-conditions L’avis est enregistré et pris en compte dans l’évaluation du prestataire.

Tableau 11 : Cas d’utilisation - Gérer les disponibilités

Élément Description
Nom du cas d’utilisation Mettre à jour les disponibilités du prestataire
Acteurs Prestataire
Déclencheur Le prestataire modifie ses paramètres de disponibilité depuis son espace personnel.
Préconditions Le prestataire est authentifié et a accès à son interface de gestion.
Description / Flux principal 1. Le prestataire accède à son espace personnel.
2. Il ouvre la rubrique relative à son profil ou à sa disponibilité.
3. Il modifie ses informations de disponibilité ou ses paramètres d’activité.
4. Il enregistre les changements.
5. Le système met à jour les données correspondantes dans la base de données.
6. Les nouvelles informations sont prises en compte dans l’affichage du profil et dans les futures réservations.
Scénarios alternatifs / Exceptions Erreur technique ou réseau : les modifications ne sont pas enregistrées.
Données invalides : le système demande une correction avant validation.
Post-conditions Les informations de disponibilité du prestataire sont mises à jour dans le système.
